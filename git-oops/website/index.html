All commands

8 commands that fix your daily Git disasters

wrong-branch

Move commits to new branch, reset current branch to upstream

git oops wrong-branch feature/fix
split

Split staged changes into separate commits by directory

git oops split --dry-run
yank

Stash changes, pull latest, restore changes safely

git oops yank
pocket

Save working state to hidden refs, optionally push

git oops pocket --push
revert-merge

Safely revert merge commits with checks and backups

git oops revert-merge abc1234
undo

Undo commits but keep changes in working directory

git oops undo -n 3
fixup

Fix the last commit with current changes

git oops fixup -m "better message"
save

Quickly save all changes with a commit

git oops save -m "WIP: feature work"

Safety rails included

Never lose work with built-in protections

Backup tags

Every destructive operation creates a safety tag like oops-backup-20240315-142301 so you can always recover.

Protected branches

Detects main/master/develop and warns before destructive operations. Configure with git config oops.protected.

Dry run mode

Use --dry-run on any command to see exactly what would happen without making changes.

Push detection

Warns if you're about to rewrite commits that have been pushed, with option to abort.

FAQ

Is this safe to use on production repos?

Yes. git-oops creates safety tags before destructive operations, detects protected branches, and warns about pushed commits. All commands support --dry-run to preview changes.

How is this different from other Git tools?

git-oops focuses on specific "oops" moments every developer faces. Instead of learning complex Git internals, you get simple commands for common mistakes with safety built in.

Do I need AI features?

No! All core commands work without AI. AI features are optional enhancements for commit messages, conflict resolution, and code analysis when you configure a provider.

What if something goes wrong?

Every destructive operation creates a backup tag. Run git tag --list "oops-backup-*" to see them, then git reset --hard TAG_NAME to restore.

Does it work with my Git workflow?

Yes. git-oops uses standard Git operations under the hood and doesn't modify your repository structure. It works with any Git workflow, hosting provider, or team setup.
